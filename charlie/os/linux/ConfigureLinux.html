Configure Linux

We frequently need to configure an Ubuntu flavored Linux box from scratch. Your instance might be:

  • Running on hardware
  • Running in VirtualBox or VmWare
  • Running in EC2
  • Running somewhere else.

In all cases, you might find the following information useful.

Note that this information applies to most varieties of Ubuntu, including:

  • Ubuntu Desktop
  • Ubuntu Server
  • Ubuntu Minimal
  • Lubuntu
  • And to a large degree, even the Ubuntu flavored Mint releases.

Update the Machine

Our first job on a new install, and one that you should perform regularly after the install, is updating the OS. It is possible to have this done automatically, and in the Desktop versions there are tools to do it through the GUI, but I frequently do it by hand from the command prompt. All you need do is issue the following two commands.

sudo apt-get update
sudo apt-get upgrade

The process listed above is simple to perform, and usually completes in only a few minutes or less.

An upgrade from on release to another, takes longer. For instance, moving from Ubuntu 13.10 to 14.04 takes a long time. That is natural, since it is a more complex process. It would be like upgrading from Windows Vista to Windows 8. Only without the cost, and usually without the hastle. Please note that on slow machines an upgrade from one release to another release can take several hours. Nevertheless, if you want to update from one version of the distro to the next version, then use this command:

do-release-upgrade

Install Git

The command is simple:

sudo apt-get install git

Then create a directory called Git:

cd
mkdir Git
cd Git

If you are one of my students, you might want to check out a read only copy of JsObjects:

git clone git://github.com/charliecalvert/JsObjects.git

Install SSH

On EC2, SSH is already installed. But on a fresh Ubuntu install, the steps are simple:

apt-get install ssh

And then create a key pair for yourself by following the instructions found here:

  • Key pair creation on Elvenware

Here is how to SSH to a remote machine:

ssh <userName>@<theServerYouWantToConnectTo>

For instance:

ssh ubuntu@192.168.1.23

This is an advanced trick, and not one that you necessarily want to use. You can use ssh-copy-id to install your public key in the authorized_keys file on a remote machine to which you already have access:

ssh-copy-id <userName>@<theServerYouWantToConnectTo>

On the server edit /etc/ssh/sshd_config and set Password Authentication to no.

Install Oracle/Sun Java

Java gets installed in the /usr/lib/jvm directory.

The new improved way:

sudo add-apt-repository ppa:webupd8team/java
sudo apt-get update
sudo apt-get install oracle-java8-installer

And you can test it like this:

java -version

The reference for the above is here.

The old, now obsolete way to install Java:

sudo apt-add-repository ppa:flexiondotorg/java
sudo apt-get update
sudo apt-get install sun-java6-jdk sun-java6-plugin
sudo update-alternatives --config java
